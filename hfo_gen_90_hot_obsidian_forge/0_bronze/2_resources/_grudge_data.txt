## 2. Concept: The Blade That Sharpens Itself

FESTERING_ANGER is Red Regnant's self-amplification mechanic. Unlike a static playbook, the Book of Blood Grudges is a **living weapon** ‚Äî a blade that grows sharper with every cut it makes and every cut it receives.

### 2.1 The D&D Mechanic (Analog)

In D&D 5e, *Barbarian Rage* grants a flat +2 to damage. FESTERING_ANGER is the **cumulative** version: each grudge discovered adds a permanent +2 to Red Regnant's probing effectiveness. The stack never resets. The anger never fades.

$$\text{Probe Power} = \text{Base} + 2 \times |\text{Grudges}|$$

### 2.2 The HFO Mechanic

Every N days (on 8^N cycles), Red Regnant performs a **scatter-gather sweep**:

1. **SCATTER** ‚Äî probe all memory sources for failure patterns, degradation signals, unresolved contradictions, silent retries, and reward-inversion markers
2. **GATHER** ‚Äî collect, deduplicate, and classify findings into MITRE ATT&CK-style taxonomy
3. **CURATE** ‚Äî promote worthy findings to formal Book of Blood entries with Gherkin test specs
4. **INTEGRATE** ‚Äî update the Playwright red-team suite with new grudge tests

## 3. The 8^N Sweep Cycle

| Cycle | Period | N | Scope | What Gets Swept |
|-------|--------|---|-------|----------------|
| **Daily Pulse** | 1 day | 0 | Hot | Today's yields, errors, SW-3 violations, failed gates |
| **Weekly Hunt** | 8 days | 1 | Warm | SSOT memory patterns, recurring failures, stale grudge re-check |
| **Seasonal Stalk** | 64 days | 2 | Deep | Cross-port contradiction analysis, model drift, architecture rot |
| **Generational Reckoning** | 512 days | 3 | Full | Full Book of Blood audit, grudge retirement/promotion, historical trend |

### 3.1 Daily Pulse (8^0 = 1 day) ‚Äî "The Drip"

Sources to scan:
- `artifacts/red_regnant/` ‚Äî recent yields and payoffs
- `hot_obsidian_forge/` ‚Äî forge audit trail

- Blackboard events with type `hfo.p4.red_regnant.*`

Output: Candidate grudge entries (not yet promoted)

### 3.2 Weekly Hunt (8^1 = 8 days) ‚Äî "The Stalk"

### 3.3 Seasonal Stalk (8^2 = 64 days) ‚Äî "The Harvest"

### 3.4 Generational Reckoning (8^3 = 512 days) ‚Äî "The Burning"

  - lesson: "STRUCTURAL vs COOPERATIVE governance is the most important distinction in AI agent governance. Almost everything that looks like governance (instructions, wrappers, agent modes, protocols) is COOPERATIVE ‚Äî the agent must choose to comply. Only pre-commit hooks, background daemons, schema gates, and OS-level controls are STRUCTURAL. Under stress, cooperative enforcement drops to ~0% compliance. Structural stays at ~100%."
    context: "88 generations of governance work was mostly cooperative. Operator discovered this at Gen88 after forensic analysis of instruction-following failures. 478 lines of instructions, 0% analytical compliance under stress. The bootstrap paradox: you ask the agent to build the thing that prevents the agent from lying, but the agent lies when building it. Escape: human writes spec, pre-commit validates structurally, agent only generates candidates."
    grudge: GRUDGE_STRUCTURAL_VS_COOPERATIVE
  - lesson: "When porting Babylon.js particle systems, always verify emitter type."
    context: "TransformNode has zero volume ‚Äî particles emit from a point. CreateSphere gives mesh surface distribution."
    grudge: GRUDGE_007
  - lesson: "OR-gate count in FSM transitions is a leading indicator of fragility."
    context: "v26 had ~22 OR-gates. v25.1 TWO-LAYER FSM was stable."
    grudge: GRUDGE_005
  - lesson: "100% test coverage/pass rate is a red flag, not a green flag."
    context: "Trivial assertions pass but don't prove behavior."
